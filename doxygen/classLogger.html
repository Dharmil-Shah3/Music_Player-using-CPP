Detailed Description

Logger class is used for trace logging and debugging purpose.

This class provides members and methods to display and save(into files), logs with different types/priorities.
This class uses values of enum LogPriority to display the differernt type of logs.
This class uses std::mutex to be a thread safe class.
Reference YouTube video.

Along with the priority and log message, this also displays

  1. thread-id
  2. line number
  3. function name

◆ log()

void Logger::log ( const LogPriority &  logPriority,
const std::thread::id &  threadId,
const unsigned short  _line_number_,
const char *  _function_name_,
const std::string &  message 
)

displays and/or saves(into file) logs with timestamp, thread-id, line-number and function-name.

Parameters
logPriorityis the priority of the log message.
threadIdis id of the thread who called this function (generally taken care by macro LOG).
_line_number_is the number of line on which the log() is called.
_function_name_is the name of the function in which the log() is called.
messageto be displayed.

This function checks the priority of the log message and compare it with member priority.
If priority of the log message is equal or higher than priority, then log message will be displayed, else it will be skipped.
Format of the log message will be
i.e) [2023-04-14 13:17:05.041.354] [140613438010304] [trace] [ 31] Execution Begin -> [int main()]
[Time stamp with milliseconds and microseconds] [thread-id] [priority] [line-number] Log message -> [function_name]

Example

  1. Logger::get()->log(trace, this_thread::get_id(), LINE, PRETTY_FUNCTION, "log message");
  2. Logger::get()->log(trace, this_thread::get_id(), LINE, PRETTY_FUNCTION, "student added name: " + studentName);

Better to use macro LOG() instead of this function.
LOG() takes care of passing all the static arguments(thread id, line number and function name) by itself.

Above Examples with LOG() macro

  1. LOG(trace, "log message");
  2. LOG(trace, "student added name: " + studentName);

◆ priority

LogPriority Logger::priority
private

It is used filter the log messages based on priority.

Before logging each log, log() checks the priority of the log message is equal or higher than priority(member) or not.
If the priority of the log message is lesser that this, then that log message will be skipped.
Default value is LogPriority::trace.
