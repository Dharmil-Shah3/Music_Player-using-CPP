)    (Logger::get()-> Logger::log(priority, std::this_thread::get_id(), __LINE__, __PRETTY_FUNCTION__, message, __VA_ARGS__))

It is used for logging, and is a short hand of the Logger::log().

Parameters
priorityis the LogPriority of the log.
messageis the log message.
...is the variable arguments to go with the message (if any). Have to pass NULL if there is no argument.

It is used as an alternative of Logger::log()
Main objective is, to pass static arguments like thread id, line number and function name into Logger::log().
This macro internally calls the log() with all the necessary arguments passing by it self.
So user have to pass only necessary parameters.

There are 2 differences between LOG() and Logger::log().

  1. Unlike log(), in LOG, if there is no extra argument after message, we have to pass NULL explicitely.
  2. Unlike log(), in LOG, we don't have to pass thread id, line number and function name explicitely.

Examples

  1. LOG(trace, "custome log 1", NULL);
  2. LOG(debug, "object id %d, object name %s", id, name);
