It is used for logging, and is a short hand of the {\ttfamily \doxylink{classLogger_ad24b3e72e63712513cf3b07cb6d4d6e0}{Logger\+::log()}}. 


\begin{DoxyParams}{Parameters}
{\em priority} & is the {\ttfamily Log\+Priority} of the log. \\
\hline
{\em message} & is the log message. \\
\hline
{\em ...} & is the variable arguments to go with the message (if any). Have to pass NULL if there is no argument.\\
\hline
\end{DoxyParams}
It is used as an alternative of {\ttfamily \doxylink{classLogger_ad24b3e72e63712513cf3b07cb6d4d6e0}{Logger\+::log()}}~\newline
Main objective is, to pass static arguments like {\bfseries{thread id}}, {\bfseries{line number}} and {\bfseries{function name}} into {\ttfamily \doxylink{classLogger_ad24b3e72e63712513cf3b07cb6d4d6e0}{Logger\+::log()}}.~\newline
This macro internally calls the {\ttfamily log()} with all the necessary arguments passing by it self.~\newline
So user have to pass only necessary parameters.

There are 2 differences between {\ttfamily \doxylink{logger_8h_af6e5e4b8e7d76867f7f11a9786011326}{LOG()}} and {\ttfamily \doxylink{classLogger_ad24b3e72e63712513cf3b07cb6d4d6e0}{Logger\+::log()}}.~\newline

\begin{DoxyEnumerate}
\item Unlike {\ttfamily log()}, in {\ttfamily LOG}, if there is no extra argument after {\ttfamily message}, we have to pass {\bfseries{NULL}} explicitely.
\item Unlike {\ttfamily log()}, in {\ttfamily LOG}, we don\textquotesingle{}t have to pass {\bfseries{thread id}}, {\bfseries{line number}} and {\bfseries{function name}} explicitely.
\end{DoxyEnumerate}

\item LOG(debug, \char`\"{}object id \%d, object name \%s\char`\"{}, id, name); 
\end{DoxyEnumerate}
